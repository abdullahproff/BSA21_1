User Story

Я как пользователь хочу, получить уведомление о степени актуальности задачи, 
чтобы я мог эффективнее выставлять приоритеты в работе.

Use Case

Основной сценарий:

1-Пользователь выполняет действия на устройстве.
2-Система выдает уведомление согласно заданным настройкам/параметрам.
3-Пользователь закрывает уведомление.
4-Система переходит в режим ожидания.
5-Пользователь продолжает свою работу.

Критерий успеха: Пользователь получил необходимое ему уведомление,
систематизировав и оптимизировав свой рабочий процесс.

@startuml

actor User as "Пользователь"
participant Frontend as "Мобильное приложение"
participant Backend as "Бэкенд"
participant Database as "База данных"

loop Периодичность опроса (Фронтенд периодически опрашивает бэкенд о наличии напоминаний)
    Frontend -> Backend : 1. http - Запрос для получения напоминаний
    activate Frontend
    activate Backend

    Backend -> Backend : 2. Проверка времени срабатывания триггера

    alt Время показа напоминаний совпадает с текущим
        Backend -> Database : 3. SQL - Запрос информации для напоминания
        activate Database

        Database --> Backend : 4. SQL - Ответом возвращают информацию
        deactivate Database

        Backend --> Frontend : 5.1. http - Ответом происходит отправка напоминания
        Frontend --> User : 6. Отображение напоминания
    else Время показа напоминаний не совпадает с текущим
        Backend --> Frontend : 5.2. Нет новых напоминаний
    end

    deactivate Backend
    deactivate Frontend
end

note right of User
    Критерий успеха:
    Пользователь получил необходимые ему напоминания.
end note

@enduml
